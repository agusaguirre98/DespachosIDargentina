# Guía de Contribución

1. Crea un branch descriptivo a partir de `main`.
2. Sigue los estilos definidos en `apps/backend/pyproject.toml` y las recomendaciones de ESLint/Prettier para el frontend.
3. Agrega pruebas o actualiza las existentes siempre que sea posible.
4. Actualiza la documentación relacionada en `docs/` o los `.md` correspondientes.
5. Ejecuta los linters y pruebas locales antes de abrir un Pull Request.
6. Completa la plantilla de PR y etiqueta a los code owners relevantes.
